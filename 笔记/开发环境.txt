长久环境变量配置(路径根据php安装路径来改)：
使用打开~/.bashrc，在末尾添加如下内容：
export PATH=/usr/local/php/bin:$PATH
export PATH=/usr/local/php/sbin:$PATH
然后执行  source ~/.bashrc
临时环境变量配置(路径根据php安装路径来改)：
执行命令
echo "PATH=$PATH:/usr/local/php7/bin">> /etc/profile
echo "export PATH">> /etc/profile
然后执行  source /etc/profile

如果在使用phpize命令报错 就全目录查找下phpize的配置文件路径 然后修改这个文件 里面的php路径可能是错误的 他默认的路径是安装php时候默认的路径 不是修改后的 改过以后应该就可以了 如果没有找到配置文件 那应该不是配置文件的问题 这时候到需要编译安装的扩展下面 然后 输入phpize的完整目录 应该就可以了

php-fpm.pid是运行php-fpm时自动生成的 如果提示缺少 先运行service php-fpm start 如果还不行 就killall php-fpm 然后在运行 php-fpm的启动脚本文件在php编译目录下的sapi/fpm里 文件init.d.php-fpm就是php-fpm的启动脚本文件

server {

	#root   /var/www/test;

    location / {
        root   /var/www/test;
        index  index.php index.html index.htm;
    }

    #error_page  404              /404.html;

    # redirect server error pages to the static page /50x.html
    #
    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   /var/www/test;
    }

    # proxy the PHP scripts to Apache listening on 127.0.0.1:80
    #
    #location ~ \.php$ {
    #    proxy_pass   http://127.0.0.1;
    #}

    # pass the PHP scripts to FastCGI server listening on 127.0.0.1:9000
    #
    location ~ \.php$ {
        root           /var/www/test; #root是项目的根目录
        fastcgi_pass   127.0.0.1:9000; # 这个要和www.conf 也就是php-fpm的端口配置一至 只有这样才能正确监听解析php文件
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME  /var/www/test$fastcgi_script_name;#这里的/var/www/test 一定要正确 项目的根目录
        include        fastcgi_params;
    }

    # deny access to .htaccess files, if Apache's document root
    # concurs with nginx's one
    #
    #location ~ /\.ht {
    #    deny  all;
    #}
}
